**2026 年 3 月 25 日**  PicoClaw 团队发布 v0.2.4 版本

这次更新很大，大到我们自己都觉得，不说清楚你可能不知道升级了什么

所以这篇文章只说一件事：这次更新对你意味着什么

---

## 一、AI 助手会"分身"了

以前你让 PicoClaw 做一件复杂的事——比如"帮我搜索最新的 AI 新闻，整理成报告，然后发给我"——

它只能一步一步来，你得等着

现在不用了

它可以同时派出多个"小助手"并行干活：

一个去搜索，一个去整理，一个去发送

主助手继续响应你，不被卡住

更重要的是：

🔍 你随时能查看每个小助手在干什么

⛔ 任务跑偏了？一句话叫停，它立刻停下来

🔄 出了问题？能回滚到之前的状态，不留烂摊子

📊 每次对话用了多少"思考量"都有记录，不会悄悄超支

这套机制叫 Sub-turn + Steering，是这次最核心的底层升级

你不需要懂技术细节，只需要知道：它现在更能干了，也更听话了

---

## 二、微信终于能用了

这是很多国内用户等了很久的功能

现在只需要在网页端点击「微信」频道：

![Web UI](../../../assets/blog/v024-release/webui.png)

或在命令行输入一行命令：

```
picoclaw auth weixin
```

![命令行](../../../assets/blog/v024-release/cli.png)

扫个码，你的微信就变成了 AI 助手的入口

发消息、收回复、发图片、发文件——和平时用微信一样，只是另一头是 AI 在回你

> ⚠️ 说清楚一件事：这个功能用的是腾讯官方接口，腾讯有权查看对话内容。PicoClaw 本身不会把你的数据发给任何第三方，但腾讯那边的权限我们管不了，用之前请自己权衡

---

## 三、企业微信也重做了

如果你在公司用企业微信，这次有个重要变化：

原来有三个独立的企业微信配置（`wecom`、`wecom_app`、`wecom_aibot`），现在合并成一个了

![企业微信](../../../assets/blog/v024-release/wecom.png)

好处是：配置更简单，功能更完整，回复支持流式输出（字一个个蹦出来，不用等全部生成完）

> ⚠️ 如果你之前配置过企业微信，升级前需要迁移配置，否则会失效

---

## 四、你的密钥更安全了

以前 PicoClaw 把所有配置——包括你的 API 密钥、Bot Token——都存在 `config.json` 里

这个文件如果不小心分享出去，密钥就泄露了

现在不一样了：

🔐 所有敏感信息单独加密存储在 `.security.yml` 里

🙈 Web 界面上的 API 密钥至少遮住 40%，不会完整显示

🚫 AI 在处理工具结果时，如果发现里面有密钥或密码，会自动替换成 `[REDACTED]`，不会把你的敏感信息喂给模型

📝 日志里的 Bot Token 只显示首尾各 4 个字符

升级后第一次启动，PicoClaw 会自动把旧配置里的敏感信息迁移过去，你不需要手动操作

## 五、支持的 AI 模型又多了

这次新增了：

| 新增 | 说人话 |
|------|--------|
| AWS Bedrock | 亚马逊云上的 AI，Claude、Llama 都能用 |
| Azure OpenAI | 微软 Azure 上的 GPT |
| 阿里云通义千问 | 国内访问快，还有专门写代码的版本 |
| Novita AI | 又一家 AI 服务商 |
| 百度千帆 AI Search | 百度的 AI 搜索，国内访问快 |

另外，现在可以给同一个模型配多个 API Key

一个 Key 挂了，自动切换到下一个，你感知不到任何中断

## 六、网页界面更好用了

以前很多配置只能改配置文件，现在大部分能在网页上点点鼠标搞定：

✅ 首页直接能聊天，不用进子菜单

✅ 所有渠道在一个页面管理，开关、配置都方便

✅ AI 调用了什么工具、执行结果如何，聊天界面里实时可见

✅ 消息支持混合排版，表格、代码、图片都能正常显示

✅ 企业微信和微信的扫码绑定，直接在网页上操作

## 七、语音功能增强了

- 支持 **ElevenLabs** 语音转文字
- 支持用任意 AI 模型（比如 GPT-4o Audio）来做语音转录
- Telegram 现在可以直接发语音消息回复

- ✅ Telegram 机器人一直显示"正在输入"——修了
- ✅ 微信有时候加载不出来——修了
- ✅ 保存 API 密钥保存不了——修了
- ✅ 某些 AI 模型回复空白——修了
- ✅ QQ 长语音文件处理失败——修了
- ✅ 飞书消息表格显示不全——修了

- ✅ 模型轮询从错误的位置开始——修了

共计 35 项 bug 修复

---

## 需要注意的事

**升级前请备份你的 `config.json`**

这次有几个配置结构变了：

1. 企业微信的 `wecom_app` 和 `wecom_aibot` 配置块已移除，需要迁移
2. 旧版 `providers` 配置不再支持，需要改用 `model_list`
3. 飞书渠道不支持 32 位老设备

如果你现在用得好好的，建议先备份再升级
